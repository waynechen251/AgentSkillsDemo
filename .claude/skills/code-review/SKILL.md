# /code-review — 程式碼審查助手

## 參數

- `$ARGUMENTS`（選填）：要審查的檔案或目錄路徑，預設為工作目錄下所有已變更的檔案

## 執行環境

此 Skill 在 **fork 子代理** 中以 **唯讀模式** 執行（Explore agent），確保審查過程不會修改任何檔案。

## 審查流程

1. **識別審查範圍**
   - 若指定路徑，審查該路徑下的所有檔案
   - 若未指定，使用 `git diff` 與 `git status` 找出所有變更檔案

2. **五維度分析**
   針對每個檔案進行以下維度的審查：

   | 維度 | 檢查重點 |
   |------|---------|
   | 🔍 正確性 | 邏輯錯誤、邊界條件、型別安全、空值處理 |
   | 🔒 安全性 | 注入風險、敏感資料曝露、權限控制、OWASP Top 10 |
   | ⚡ 效能 | 不必要的迴圈、記憶體洩漏、N+1 查詢、快取機會 |
   | 🛠️ 可維護性 | 命名規範、程式碼重複、函式長度、耦合度 |
   | 🧪 測試覆蓋 | 缺少測試、邊界測試、模擬適當性 |

3. **產生審查報告**

## 輸出格式

```markdown
# 程式碼審查報告

## 總覽
- **審查範圍**：<檔案數量與路徑摘要>
- **整體評級**：⭐⭐⭐⭐☆（4/5）
- **關鍵發現**：<最重要的 1-3 項發現>

## 詳細審查

### 📄 <檔案路徑>

#### 🔍 正確性
- ✅ <良好的部分>
- ⚠️ <需注意的部分>（行號：建議）

#### 🔒 安全性
...

#### ⚡ 效能
...

#### 🛠️ 可維護性
...

#### 🧪 測試覆蓋
...

---

## 建議優先處理項目
1. 🔴 **嚴重**：<描述>
2. 🟡 **建議**：<描述>
3. 🟢 **可選**：<描述>
```

## 注意事項

- 此 Skill 為唯讀操作，不會修改任何檔案
- 若審查範圍過大（超過 20 個檔案），會提示使用者縮小範圍
- 審查結果使用繁體中文撰寫
